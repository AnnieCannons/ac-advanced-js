Strings

Strings as Variables

Upto this point, you are familiar with two ways to create strings using ES5 syntax.

  • Using single quotes: 
    
var str1 = 'Hello';
  • Using double quotes: 
    
var str2 = "Hello";

ES6 adds a third way for us to create strings.

  • Using backticks (`): 
    
var str3 = `Hello`;

Review: ES5 Concatenation

Concatenation in ES5 joins one string with another using the + sign. In JS Bin, create two strings, concatenate them, and log their concatenation to the console.


//This method does not involve storing the strings in variables.
console.log("Hello" + "World");

Let's say you stored the above strings as well as their concatenation in each of their own variables. How would you log their concatenation to the console?

                       
var str1 = "Hello";
var str2 = "World";
var concat = str1 + str2;
console.log(concat);

ES6 Concatenation

In ES6, we can use backticks (`) to concatenate.
Find the ` on your keyboard.

Using backticks to replace the quotes for concatenation, please log the two strings "Hello" and "World" to the console.


//This method does not involve storing the two strings in variables.
console.log(`Hello` + `World`); //"HelloWorld"

When joining strings that are only strings and not variables, the + sign can be dropped.


console.log(`HelloWorld`); //"HelloWorld"

Even better: we can easily include a space between words.


console.log(`Hello World`); //"Hello World"

String Concatenation with Variables

Using ES6 syntax to insert or interpolate variables into a string is called string interpolation.

A $ followed by curly braces { } that wrap around the variable's name permits its insertion or interpolation into a string. Example: ${variableName}

String interpolation is complete when we insert the variable into a pair of backticks. Example: `${variableName}`

Given a var name = "WeMac"; use backticks to log a string containing the variable to the console.


console.log(`${name}`); //"WeMac"

String Interpolation

Now that you have inserted a variable into a string, expand your string with a greeting to WeMac that reads:
"Hey, WeMac!"


console.log(`Hey, ${name}!`); //"Hey, WeMac!"

How would you add an additional variable? Perhaps, have the console print: "Hey, [cohort name], Class of [year]!"


var year = 2020;
console.log(`Hey, ${name}!, Class of ${year}!`); //"Hey, WeMac, Class of 2020!"

Notice that number data types inserted into strings also become strings in ES6, just like in ES5.

Unary Plus (+)

The plus sign (+) can be used as a unary operator.

A unary operator performs an operation on a single operand.

A unary plus (+) precedes the operand, meaning it is positioned before the operand: +value

The unary plus tries to convert the operand into a number. Example: if the value is a string of "2", then it will return a number of 2.

Unary Plus (+) Example

In place of parseInt(), how would you use the unary plus?


var age = parseInt(prompt("What is your age?"));

We drop the parseInt() and put the unary plus (+) before the prompt().


var age = +prompt("What is your age?");

More on the Unary Plus

When the unary plus is given different kinds of values to convert to numbers, it can return some interesting values.


+3                                   // returns 3
+'-3'                                // returns -3
+'3.14'                              // returns 3.14
+'3'                                 // returns 3
+'0xFF'                              // returns 255
+true                                // returns 1
+'123e-5'                            // returns 0.00123
+false                               // returns 0
+null                                // returns 0
+'Infinity'                          // returns Infinity
+'infinity'                          // returns NaN
+function(val){  return val }        // returns NaN

Booleans

Booleans as Variables

We have learned that there are only two values for a boolean: true or false.

Booleans can be easily interpreted and compiled into machine code of 1's or 0's, on or off, yes or no,
run the program or not to run the program.

Booleans are excellent for programs!

Booleans in Programs

  • loops
    • If a condition evaluates to true, the loop will run.
    • If a condition evaluates to false, the loop will stop.
  • if/else statements
    • If the if condition evaluates to
      • true, the if block runs.
      • false, the if block is ignored.
    • If the if condition is false, the program will evaluate the else if condition (if the else if statement exists).
      • If true, the else if block runs.
      • If false, the else if block is ignored.
    • If if/ else if is false, the program runs the else block.

Ternary Operator

Up to this point we have been seeing two kinds of operators:

  • unary operator (e.g., unary plus)
  • binary operator (e.g., equality operator)

Unlike a unary operator that takes in one operand and a binary operator that takes in two operands,
a ternary operator takes in three operands.

if/else statements are ternary:

  • Operand 1: the condition to evaluate as true or false
  • Operand 2: the expression to run if true
  • Operand 3: the expression to run if false

Ternary Operator

The ternary operator for if/else statements uses this syntax:

condition ? expressionIfTrue : expressionIfFalse


//Refactor me into a ternary statement!
var weather = "sunny";
if (weather === "sunny") {
    console.log("Beach time!");
} else {
    console.log("Hot cocoa time!");
}

Identify the condition to evaluate.


(weather === "sunny")

Insert the ? for the ternary statement.


(weather === "sunny") ?

Ternary Operator

Expression If True


//Refactor me into a ternary statement!
var weather = "sunny";
if (weather === "sunny") {
    console.log("Beach time!");
} else {
    console.log("Hot cocoa time!");
}

So far, our refactoring looks like this:


(weather === "sunny") ?

Next, we identify the expression to run if the condition evaluates to true and insert after the ?.


(weather === "sunny") ? console.log("Beach time!")

Ternary Operator

Expression if False


//Refactor me into a ternary statement!
var weather = "sunny";
if (weather === "sunny") {
    console.log("Beach time!");
} else {
    console.log("Hot cocoa time!");
}

So far, our refactoring looks like this:


(weather === "sunny") ? console.log("Beach time!")

Next, we insert a : followed by the expression to run if the condition evaluates to false.


(weather === "sunny") ? console.log("Beach time!") : console.log("Hot cocoa time!");

Ternary Operator


(weather === "sunny") ? console.log("Beach time!") : console.log("Hot cocoa time!");

We have a ternary statement! However, to see the values in console, we must save to a variable before logging.


var print = (weather === "sunny") ? console.log("Beach time!") : console.log("Hot cocoa time!");
console.log(print);

Since we are logging print to the console, we can drop the console.log() in the expressions.


var print = (weather === "sunny") ? "Beach time!" : "Hot cocoa time!";
console.log(print);

Array.prototype.map()

From Number to Number

.map() loops through each item in the input array,
runs a provided function that applies to each input item,
and outputs a new array of the returned items.

                        
const numbers = [1, 2, 3, 4];
const doubled = numbers.map(item => item * 2);
console.log(doubled); // [2, 4, 6, 8]

item is a name for each item in the input array of numbers.

The arrow function is doubling the value of each item in the input array of numbers.

The output array is stored in the new variable doubled.

Array.prototype.map()

From Number to String

.map() loops can change the data type of the input item
(per the code of the function that applies to each input item)
to become a different data type for the output array.


//this example is helpful for giving key numbers to <li>                        
const keys = [1, 2, 3, 4];
const keysUnique = keys.map(key => "key" + key);
console.log(keysUnique); // ["key1", "key2", "key3", "key4"]

key is a name for each item in the input array of keys.

The arrow function is adding a string of "key" to each key in the input array of keys.

The output array is stored in the new variable keysUnique.

Array.prototype.filter()

.filter() loops through each item in the input array,
tests each item against a provided function,
and outputs a new array of items that passed the test.


// this example works with numbers          
const numbers = [1, 2, 3, 4];
const evens = numbers.filter(item => item % 2 === 0);
console.log(evens); // [2, 4]

item is a name for each item in the input array of numbers.

The arrow function is testing if each item in the input array of numbers is evenly divisible by 2.

The output array is stored in the new variable evens.

.splice() loops through the array
and can remove or replace or add items by index position.

  1. index position of the item
  2. how many elements to remove/replace

const toRead = ["A Christmas Carol", "Becoming", "Committed"];

.splice() can accept more than 2 params.

The 3rd, 4th, 5th, etc. optional params in a .splice() are additions of items at the index position.


const nums = [1, 3, 4, 5, 6]; //use .splice() to add 2 to the nums array


nums.splice(1, 0, 2); //note the third param

//drop 0 items,
//add the item 2.

// add 2 items: "Oct" and "Nov"

Array.prototype.sort()

.sort() sorts an array using three steps:

  1. converts each item in the array into a string
  2. compares their sequences of UTF-16 code units values
  3. has default ascending sort order (based upon UTF-16)

const months = ["November", "December", "January", "February"]; 


months.sort();
console.log(months); //["December", "February", "January", "November"]

How will .sort() impact numbers?

Array.prototype.sort()

//.sort() converts the items into strings for UTF-16 comps + sequencing
//then .sort() gives you the newly ordered items in the array

So how do we .sort() number data types so that months is [1, 2, 11, 12]?

Array.prototype.sort()

Compare Function

Built into .sort() is a default compare function.


function compare(a, b) { //a and b are UTF-16 characters being compared.
  var value = (a < b) ? -1
  : (a > b) ? 1
  : 0 //when (a === b)
  return value;
} //this default function will always return numbers out of numerical order.

To sort numbers, we must pass in a function to override the default compare function.


function compareNums(a, b) { //a, b are params for UTF-16 comps, sequencing.
    return a - b;
} //this override function will always return numbers in numerical order.

How do we pass in this override function into .sort()?

Array.prototype.sort()

We want to pass in this override function into .sort().


//Refactor this override function into an arrow function.
function compareNums(a, b) {
    return a - b;
}
//Type the code for the arrow function here:

Pass the arrow function into the .sort().


const months = [11, 12, 1, 2];
months.sort((a,b) => a - b);
console.log(months); //[1, 2, 11, 12]

Viola! The array's items are in numerical order.
